# [ BPA 시스템 운영 매뉴얼 ]
## 1. 개요
* 메인 URL: https://jwchob-lab.github.io/bpa/
* 기본 페이지 : index.html 와 links.json 파일을 통해 통합 관리
* 프로그램 : ./app 디렉토리에서 관리
* DB : ./db 디렉토리에서 관리
* 별도 설치 없이 브라우저에서 즉시 실행 가능
* GitHub 사용 : 리포지토리 용량: 1GB, 빌드 시간당 10회, 월간 대역폭 100GB
* 기술 스택: HTML 기반이며 기능에 따라 PyScript, JavaScript, AI 등을 활용
* 데이터베이스: DB가 필요한 경우 깃허브와 연동된 Supabase를 호출
## 2. 구분
### 2.1. 일반 (General)
> 디스크 사용량 예측
* 월 단위 디스크 Full 예상 그래프 생성
* TXT 붙여넣기 또는 CSV 파일 업로드 지원
* 현재 월까지의 디스크 증가량 입력 필수
> 디스크 사용량 예측 (ai)
* 제미나이(Gemini) API 활용 그래프 생성
* 이미지 캡처 붙여넣기 기능 지원 (GB 기준)
* 이슈: 이미지 파일 인식 시 실제보다 낮은 값으로 인식되는 현상 존재
> 보고서 작성 (ai)
* 제미나이 API 기반 텍스트 자동 보정 기능
> SQL 튜닝 (ai)
* 제미나이 API 를 호출하여 튜닝 가이드 수행
---
### 2.2. 데이터베이스 (DB)
> DB: 트리거
* 배포 수행시 트리거 관련 테이블이 있는지 사전 탐지 목적
* 실행: 1번(배포파일) 및 2번(테이블 목록) 창에 파일을 각각 Drag & Drop
> DB: 점검 (ai)
* 기본 점검 SQL 문 생성
* 실행: DBMS 종류 및 버전 선택후 실행
* > DB: SQL 튜닝 (ai)
* AI 에게 튜닝 가이드 요청
* 쿼리 결과가 틀리게 나오는 경우도 많아서 참고용으로만 사용 필요
### 2.3. 오라클 (Oracle)
> Oracle: XPLAN
* 유저명 및 튜닝 쿼리 입력을 받아서 XPLAN 생성 쿼리 생성
> Oracle: 권한백업
* 테이블 목록 입력을 받아서 테이블 권한 백업 쿼리 생성
---
### 2.4. 게시판
> 자유게시판
* DB 연동이 가능한지 테스트 용도
* 깃허브에서 Supabase DB 호출 가능하도록 연동
* 환경: PostgreSQL 17.6 (aarch64-linux, gcc 15.2.0)
* 주의: 실제 운영 시 보안 파일 분리 및 DB 권한 설정 보완 필요
