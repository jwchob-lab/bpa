# [ BPA 시스템 운영 매뉴얼 ]
## 1. 개요
* 메인 URL: https://jwchob-lab.github.io/bpa/
* 웹 기반 실행: 별도 설치 없이 브라우저에서 즉시 실행 가능합니다.
* GitHub 배포: 각 프로그램은 깃허브 레파지토리를 통해 배포됩니다. (GitHub)
* 기술 스택: HTML 기반이며 기능에 따라 PyScript, JavaScript, AI 등을 활용합니다.
* 링크 관리: links.json 파일을 통해 통합 관리됩니다.
* 데이터베이스: DB가 필요한 경우 깃허브와 연동된 Supabase를 호출합니다. (Supabase)
<br>
<br>
## 2. 구분
### 2.1. 일반 (General)
> 트리거 파악
* 배포 전 트리거 파악 프로그램 배포 및 내부 공유 (공통 DBA 4인)
* 실행: 1번(배포파일) 및 2번(테이블 목록) 창에 파일을 각각 Drag & Drop
> 디스크 관리
* 월 단위 디스크 Full 예상 그래프 생성
* TXT 붙여넣기 또는 CSV 파일 업로드 지원
* 현재 월까지의 디스크 증가량 입력 필수
> 디스크 관리 (AI)
* 제미나이(Gemini) API 활용 그래프 생성
* 이미지 캡처 붙여넣기 기능 지원 (GB 기준)
* 이슈: 이미지 파일 인식 시 실제보다 낮은 값으로 인식되는 현상 존재
> 보고서 작성 (AI)
* 제미나이 API 기반 텍스트 자동 보정 기능
---
### 2.2. 오라클 (Oracle)
> 권한 백업
* 테이블 목록 입력 시 테이블 권한 백업 자동 실행
> XPLAN
* 유저명 및 튜닝 쿼리 입력 시 XPLAN 생성문 자동 추출
---
### 2.3. 데이터베이스 (DB)
> 자유게시판
* 깃허브 - Supabase DB 호출 연동
* 환경: PostgreSQL 17.6 (aarch64-linux, gcc 15.2.0)
* 주의: 실제 운영 시 보안 파일 분리 및 DB 권한 설정 보완 필요
